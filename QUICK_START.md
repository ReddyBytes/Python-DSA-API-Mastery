## How Each Mastery Repo Is Organised

Every mastery follows the same pattern:

```
theory.md           → Core concept explained clearly
visual_explanation.md → Diagrams, step-by-step traces
cheatsheet.md       → Quick reference, templates, syntax
patterns.md         → How to recognise the problem type
common_mistakes.md  → What goes wrong and how to fix it
real_world_usage.md → Where this appears in production systems
interview.md        → Q&A pairs to practise
*.py files          → Hands-on code to run and modify
```

Not every topic has all files. But `theory.md` and `interview.md` exist everywhere.

Read in this order for each topic:
```
theory → visual → patterns → real_world → common_mistakes → cheatsheet → interview
```

## What Not to Do

- Do not read randomly. Follow the order.
- Do not just read. Write code from scratch.
- Do not skip complexity analysis. It is the foundation.
- Do not go to advanced DSA before mastering arrays, strings, and hashing.
- Do not try to finish everything fast. Depth beats speed.
