# 🔭 Observability — Theory

> Know what your system is doing in production — metrics, tracing, logging, and alerting for reliable systems.

---

## 📋 Contents

```
1.  The three pillars — metrics, tracing, and logs
2.  Metrics types — counters, gauges, and histograms
3.  Distributed tracing — spans, trace context, and OpenTelemetry
4.  Structured logging — JSON logs, correlation IDs, log levels
5.  Alerting — thresholds, anomaly detection, and on-call hygiene
6.  SLOs and error budgets — data-driven reliability targets
7.  Dashboards with Grafana — visualising system health
8.  ELK stack — Elasticsearch, Logstash, and Kibana
```

## 📌 Learning Priority

**Must Learn** — Core concept, daily use, interview essential:
three pillars (metrics/traces/logs) · distributed tracing (spans/trace context) · SLO and error budget concepts

**Should Learn** — Important for real projects, comes up regularly:
metrics types (counter/gauge/histogram) · structured log correlation · alerting thresholds

**Good to Know** — Useful in specific situations, not always tested:
OpenTelemetry instrumentation · Grafana query basics · ELK stack overview

**Reference** — Know it exists, look up syntax when needed:
metric cardinality explosion · sampling strategies · alert fatigue management
