# 🎙️ Interview Framework — Theory
> A repeatable 45-minute structure for tackling any system design problem confidently and systematically.

---

## 📋 Contents

```
1.  RESHADED framework — the 8-step interview structure
2.  Clarifying requirements — functional vs non-functional
3.  Capacity estimation — storage, throughput, and bandwidth
4.  High-level design — components and data flow
5.  Deep dive — the interviewer's chosen component
6.  Trade-offs discussion — justifying every decision
7.  Common mistakes — what separates junior from senior answers
```

---

## 📌 Learning Priority

**Must Learn** — Core concept, daily use, interview essential:
structured interview approach (requirements→estimation→design→trade-offs) · time allocation per phase · clarifying questions

**Should Learn** — Important for real projects, comes up regularly:
trade-off articulation · common mistakes to avoid · candidate communication style

**Good to Know** — Useful in specific situations, not always tested:
how to handle unknown requirements · drawing diagrams effectively

**Reference** — Know it exists, look up syntax when needed:
RESHADED framework · preparation checklist · post-design Q&A patterns
