## 📌 Learning Priority

**Must Learn** — Core concept, daily use, interview essential:
Virtual environments (`venv`) · `pip` + `requirements.txt` · `pyproject.toml` basics · PEP 8 style guide · Type hints in production code

**Should Learn** — Important for real projects, comes up regularly:
`pre-commit` hooks (Black, Ruff, mypy) · `pytest` in CI · Semantic versioning · `python_requires` metadata · Editable installs (`pip install -e .`)

**Good to Know** — Useful in specific situations:
`tox` / `nox` · `py.typed` marker · Type stubs (`.pyi`) · `bandit` / `safety` security scanning · Private package repos

**Reference** — Know it exists, look up when needed:
Binary wheels for C extensions · `MANIFEST.in` · LTS versioning · Monorepo patterns
